Como Resolver Conflitos no Git

O que são conflitos?

Conflitos acontecem quando alterações diferentes são feitas no mesmo trecho de código em branches diferentes e o Git não consegue mesclar automaticamente.

Passo 1: Identificar o conflito

Ao tentar fazer merge ou pull, o Git vai indicar arquivos com conflitos. Eles aparecem com marcações como <<<<<<< e >>>>>>>.

Passo 2: Resolver manualmente

Edite os arquivos conflitantes, escolhendo qual alteração manter ou combinando ambas.

Passo 3: Adicionar e commitar

Depois de resolver, adicione os arquivos e faça um commit:

git add 
git commit -m "fix: resolver conflitos"

Passo 4: Continuar o fluxo

Depois de resolver, continue o merge, pull ou rebase normalmente.

Dicas para evitar conflitos

  • Atualize sua branch com frequência a partir da develop/main.
  • Comunique mudanças importantes à equipe.
  • Faça commits pequenos e frequentes.
