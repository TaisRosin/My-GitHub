Conceitos Fluxo Git Conflitos

GIT

O que é o Git?

Git é um sistema de controle de versão distribuído, usado para gerenciar alterações em arquivos e projetos de forma colaborativa.

Função principal

Permite que vários desenvolvedores trabalhem no mesmo projeto sem sobrescrever o trabalho uns dos outros.

Principais características

  • Versionamento: Registra todas as mudanças do projeto, permitindo voltar a versões anteriores.
  • Branching (ramificações): Cria linhas paralelas de desenvolvimento para testar novas funcionalidades sem afetar a versão principal.
  • Commit: Registro de alterações feitas no código com mensagem descritiva.
  • Merge: Une alterações de diferentes branches.
  • Distribuído: Cada desenvolvedor tem uma cópia completa do repositório, não apenas uma versão parcial.

GITHUB

O que é o GitHub?

GitHub é uma plataforma online que hospeda repositórios Git, permitindo colaboração remota, revisão de código e gerenciamento de projetos.

Função principal

Facilitar a colaboração, armazenamento e compartilhamento de projetos Git pela internet.

Principais recursos

  • Repositórios remotos: Armazenamento central de projetos Git para acesso online.
  • Pull Requests: Solicitações de revisão de alterações antes de integrar ao projeto principal.
  • Issues: Sistema de acompanhamento de tarefas, bugs e ideias.
  • Actions: Automatização de tarefas, como testes e deploy.
  • Fork e Clone: Permitem copiar repositórios para modificar sem afetar o original.

Diferença entre Git e GitHub

Git: é a ferramenta de controle de versão que você instala no computador e gerencia mudanças localmente.

GitHub: é a plataforma online que hospeda repositórios Git, permitindo colaboração e integração entre desenvolvedores de qualquer lugar.
