Fluxo do Git e GitHub

1. Criação de branch

Para iniciar uma nova feature, criamos uma branch a partir da branch principal (geralmente develop ou main).

2. Trabalhando na branch

Fazemos commits frequentes com mensagens claras, seguindo boas práticas de commits semânticos.

3. Atualizando a branch

Antes de finalizar, atualizamos nossa branch com as últimas alterações da develop para evitar conflitos.

4. Pull Request

Quando a feature estiver pronta, abrimos um Pull Request para integrar nossa branch de feature na develop.

5. Revisão e Merge

A equipe revisa o código, aprova o PR e realiza o merge, integrando a nova funcionalidade ao projeto.

6. Deleção da branch

Após o merge, a branch de feature pode ser deletada tanto local quanto remotamente, mantendo o repositório limpo.
